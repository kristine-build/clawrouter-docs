# 鉴权体系說明（Auth）

后台管理介面鉴权方式与權限级别說明

## [說明](#說明)

后台管理介面采用多级鉴权机制，常见为：**公开**、**使用者**、**管理员**、**Root**。

## [认证方式（二选一）](#认证方式二选一)

### [Session](#session)

通过登录介面取得 Session：

- `POST /api/user/login`

### [Access Token（推荐）](#access-token推荐)

在請求头中携带：

```
Authorization: Bearer {token}
```

Token 可在「个人设置 - 安全设置 - 系统存取令牌」中生成。

## [必需請求头](#必需請求头)

部分介面要求携带使用者标识請求头：

```
New-Api-User: {user_id}
```

其中 `{user_id}` 必须与当前登录使用者匹配。

## [權限级别](#權限级别)

- **公开（Public）**：无需鉴权
- **使用者（User）**：需要登录或 Access Token
- **管理员（Admin）**：需要管理员權限
- **Root**：最高權限
