# 鑑權体系說明（Auth）

後台管理介面鑑權方式与權限级别說明

## [說明](#說明)

後台管理介面采用多级鑑權机制，常见为：**公开**、**使用者**、**管理员**、**Root**。

## [认证方式（二选一）](#认证方式二选一)

### [Session](#session)

透過登入介面取得 Session：

- `POST /api/user/login`..

### [Access Token（推荐）](#access-token推荐)

在請求头中携带：

```
Authorization: Bearer {token}
```..

Token 可在「个人設定 - 安全設定 - 系統存取令牌」中生成。

## [必需請求头](#必需請求头)

部分介面要求携带使用者标识請求头：

```
New-Api-User: {user_id}
```..

其中 `{user_id}`.. 必須与当前登入使用者匹配。

## [權限级别](#權限级别)

- **公开（Public）**：無需鑑權
- **使用者（User）**：需要登入或 Access Token
- **管理员（Admin）**：需要管理员權限
- **Root**：最高權限
